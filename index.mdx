# The operating system for client work

Client work is scattered. Code lives in GitHub, conversations in Slack,
meetings on Google Calendar, decisions in email threads. Nobody — not you,
and certainly not your client — has one place that knows the full state of a
project. Answering "where are we?" costs a meeting.

Relay connects the tools you already use and becomes the layer that watches
everything flow through them.

```text
signal (meeting, commit, message, calendar event)
      │  via integrations
      ▼
tracked event on a client's project timeline
      │
      ▼
knowledge engine ingests → source of truth
      │
      ├──▶ AI drafts next actions → you approve → pushed to your tools
      └──▶ client portal + Relay AI answers from the knowledge base
```

## What Relay does

<CardGroup cols={2}>
  <Card title="Connect" icon="plug">
    Integrations with the tools of the trade: GitHub, GitLab, Bitbucket,
    Slack, Discord, and SMTP email — with webhooks flowing in both
    directions.
  </Card>
  <Card title="Track" icon="timeline">
    Every event, todo, decision, and progress marker is captured against a
    specific client and project, forming a living timeline.
  </Card>
  <Card title="Know" icon="brain">
    The knowledge engine ingests all of it into a per-project knowledge
    base — the source of truth for the entire project lifecycle, with
    citations.
  </Card>
  <Card title="Share" icon="globe">
    A client-facing portal at `relay.<yourcompany>.com` where clients see
    status and ask the Relay AI chatbot anything about their project.
  </Card>
  <Card title="Approve" icon="check-double">
    AI drafts, humans decide. Work only lands in external tools — issues
    created, emails sent — after explicit human approval.
  </Card>
  <Card title="Self-host" icon="server">
    The runtime kernel materializes your whole stack from declarative
    manifests: modules, integrations, secrets, and docker compose.
  </Card>
</CardGroup>

## What Relay is not

- Not another project-management tool competing with Jira or ClickUp.
- Not a chat app or a code host.
- Relay is the **orchestration and knowledge layer over** those tools.

## The pieces

| Piece | What it is |
|---|---|
| Desktop app | The founder control plane (Tauri + React) |
| Backend | NestJS modular monolith — the system of record |
| Knowledge engine | pgvector retrieval — the source of truth for every answer |
| Client portal | Where your clients see progress and talk to Relay AI |
| Runtime + CLI | The self-hosting kernel: manifests, secrets, compose, lifecycle |
