## Prerequisites

- **Node.js** ≥ 20 and **pnpm** ≥ 9
- **Docker** — running, for Postgres (pgvector) and the backend container
- **Rust toolchain** (`cargo`, `rustc`) — only for the native desktop app;
  skip it if you run the UI in the browser

## Run the stack

<Steps>
  <Step title="Clone and install">
    ```bash
    git clone git@github.com:termdx/Relay.git
    cd Relay
    pnpm install
    ```
  </Step>
  <Step title="Start everything">
    ```bash
    pnpm dev
    ```

    This boots the whole local stack: the runtime daemon (`:51720`), a
    pgvector Postgres + the NestJS backend (`:3000`) as containers, and the
    desktop app. First run builds the backend image and the Rust shell —
    later runs are cached.

    Prefer the browser over a native window (no Rust build)?

    ```bash
    pnpm dev:web    # desktop UI at http://localhost:5173
    ```
  </Step>
  <Step title="Create your owner account">
    On first launch the app shows **Set up your workspace** — name, email,
    password. This is the owner account of your self-hosted instance; you
    land on the Clients page right after.
  </Step>
  <Step title="Create a client and a project">
    Add your first client, then a project on their page. The project is
    where the timeline, todos, decisions, meetings, and knowledge live.
  </Step>
  <Step title="Invite the client to their portal">
    On the client's page, click **Copy sign-in link**. Relay mints a
    single-use, 15-minute link you can paste into any channel — no email
    setup required.

    Or start the portal and let them request their own link by email:

    ```bash
    pnpm portal:dev    # client portal at http://localhost:5174
    ```

    Their portal shows live progress, pending approvals, and **Ask** — the
    Relay AI chat grounded in the project's knowledge base.
  </Step>
</Steps>

## What runs where

| Surface | URL |
|---|---|
| Desktop UI (web mode) | `http://localhost:5173` |
| Backend API | `http://localhost:3000` (`/health`) |
| Client portal | `http://localhost:5174` |
| Runtime daemon | `http://127.0.0.1:51720` (`/health`) |

**Stopping:** Ctrl-C in the dev terminal tears down what it started.

## Real AI answers

The default AI provider is an offline stub. For real drafts, embeddings,
and chat answers, add a Gemini provider to the workspace — from the desktop
(**Runtime → AI**) or the CLI (`relay ai add gemini`). The key goes into the
encrypted secret store and is injected when the runtime regenerates the
stack.

Iterating with `pnpm dev:local` instead? Env passthrough works there:

```bash
AI_PROVIDER=gemini GEMINI_API_KEY=... pnpm dev:local
```

<Tip>
  `pnpm dev:local` runs Postgres standalone and `nest start` directly —
  no image rebuilds while you edit the backend.
</Tip>
